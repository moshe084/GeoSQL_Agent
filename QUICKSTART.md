## ğŸ› Troubleshooting

### Problem: "Database connection failed"

**Solution:**
```bash
# Wait for PostGIS to initialize (can take 10-30 seconds)
docker-compose logs -f postgis

# Look for: "database system is ready to accept connections"
```

### Problem: "OpenAI API error"

**Check your API key:**
```bash
# Verify .env has valid key
cat .env | grep OPENAI_API_KEY

# Should show: OPENAI_API_KEY=sk-...
```

**Common mistakes:**
- Key doesn't start with `sk-`
- Extra spaces in `.env` file
- Key has expired or no credits

### Problem: Port already in use

**Solution:**
```bash
# Change ports in docker-compose.yml
# Default ports: 5432 (db), 8000 (api), 3010 (frontend)

# Or stop conflicting services:
docker ps  # find conflicting container
docker stop <container-id>
```

### Problem: Frontend shows but queries fail

**Check backend logs:**
```bash
docker-compose logs -f backend
```

**Verify network connectivity:**
```bash
# From frontend, API should be reachable
curl http://localhost:8000/health
```

## ğŸ“Š Understanding the Results

When you execute a query, you'll see:

1. **SQL Console** (left panel, bottom):
   - The actual PostGIS SQL query generated by AI
   - Learn PostGIS syntax by reading these!

2. **Map** (right panel):
   - Blue markers = cafes
   - Purple polygons = parks
   - Purple lines = roads
   - Click any marker for details

3. **Stats** (bottom right of map):
   - **Results:** Number of features returned
   - **Execution Time:** How long the query took
